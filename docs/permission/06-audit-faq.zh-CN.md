## 06. 日志 / 白名单 / FAQ

[English](05-audit-faq.md) | 简体中文

### 日志

- **登录日志**：记录认证登录相关行为。
- **操作日志**：记录系统关键操作事件，便于审计与追踪。

排查建议：

- **401**：优先检查登录态、Token 是否有效、是否携带正确请求头。
- **403**：检查角色是否绑定对应 API、请求是否命中白名单、Path/Method 是否匹配。

### 白名单

白名单规则在命中（Method + AntPath）时可跳过“角色-API 绑定”校验。建议只用于必要场景，并定期审计。

### 常见问题（FAQ）

- **修改了 `MYSQL_DATABASE` 但数据库名没变**
  - MySQL 初始化库只在首次初始化生效；如需重建需 `docker compose down -v`（会清空数据）。
- **页面菜单能看到但接口返回 403**
  - 菜单权限与 API 权限是两套配置，需要把对应 API 绑定到角色。
