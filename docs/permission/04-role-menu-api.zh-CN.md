## 04. 角色与授权（菜单 / API / RBAC）

[English](04-role-menu-api.md) | 简体中文

### 目标

创建角色并进行授权配置，使用户具备相应页面入口（菜单）与接口访问能力（API/RBAC）。

### 操作步骤（管理端）

1. **创建角色**
- 建议使用清晰的命名（例如：`TENANT_ADMIN`、`OPS_READONLY`）。

2. **配置菜单权限**
- 将菜单绑定到角色。
- 验证“我的菜单树”是否包含已授权菜单。

3. **配置 API 权限（RBAC）**
- 确保所需 API 已存在（Method + Path）。
- 将 API 绑定到角色。

4. **配置数据域（Data Domain）**
- 在“系统管理 / 数据域”中维护数据域（如：仅本人、本组织、包含下级、本租户全部等）。
- 在“角色管理”中将数据域绑定到角色。
- 数据域会影响列表/查询类接口返回的数据范围（数据权限）。

5. **给用户分配角色**
- 将一个或多个角色绑定给用户。

### 验证清单

- [ ] 用户可见期望的菜单入口
- [ ] 调用受保护接口返回 200（而不是 403）
- [ ] 数据列表仅返回期望范围内的数据（数据域生效）

### 注意事项

- API 匹配按 **HTTP Method + AntPath** 执行，避免随意使用过宽的 `/**` 规则。
- 数据域属于**数据范围控制**，通常与组织归属（User.org）一起使用；若未绑定数据域，可能导致看不到任何数据或范围过大（取决于系统默认策略）。
